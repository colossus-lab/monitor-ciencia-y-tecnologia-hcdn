# Contribuyendo a Leyes IA

¡Gracias por mostrar interés en colaborar con **Leyes IA**, un proyecto de **Colossus Lab**! Estamos muy felices de sumar talento a nuestra comunidad open source.

## Modos de Colaboración

Hay muchas formas en las que puedes ayudarnos:

- **Reportando bugs**: Si el dashboard falla o el chat de IA omite respuestas.
- **Sugiriendo nuevas funcionalidades**: ¿Hay algún dato extra o filtro que te gustaría ver?
- **Actualizando datos**: Contribuyendo con nuevos proyectos de ley que no hayan sido escaneados.
- **Enviando Pull Requests**: Arreglando issues conocidos o aportando nuevas mejoras.

## Cómo Reportar un Problema (Issue)

1. Revisa [nuestra lista de Issues](https://github.com/TODO_URL/issues) para asegurarte que nadie lo reportó antes.
2. Crea un nuevo Issue describiendo claramente el problema esperado *vs* el problema real.
3. Agrega pasos para reproducirlo, versión de sistema operativo y navegador.

## Sugiriendo Nuevas Funcionalidades

Los aportes son bienvenidos, pero para funcionalidades grandes preferimos que abras un Issue de tipo *Feature Request* primero. Así evitamos duplicidad de esfuerzo y nos ponemos de acuerdo en el diseño y arquitectura antes de lanzar el código.

## Proceso de Desarrollo (Pull Requests)

1. Haz un **[Fork](https://docs.github.com/en/get-started/quickstart/fork-a-repo)** del repositorio.
2. Crea una rama desde `main` (`git checkout -b feature/nueva-grafica`).
3. Realiza tus cambios y verifica que no rompen el pipeline existente (`npm run dev:static`).
4. Haz **Commit** de tus cambios usando mensajes descriptivos (e.g., `feat: agrega gráfico de torta categorizando proyectos`).
5. Sube tu rama a tu fork (`git push origin feature/nueva-grafica`).
6. Abre un **Pull Request** a nuestra rama `main`.

Asegúrate de describir tu Pull Request en detalle y referencia cualquier Issue que estés resolviendo.

¡Una vez más agradecemos tu soporte y apoyo al ecosistema argentino de datos abiertos de inteligencia artificial y desarrollo legislativo!
